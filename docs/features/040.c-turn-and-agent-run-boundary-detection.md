## 1. Value delivered

After this slice ships, `turn_id` is stable across continuation round-trips of one user turn (multi-tool-use loops share one `turn_id` until `stop_reason ≠ tool_use`), and `agent_run_id` is stable across all turns of one agent persona within a session (boundary changes on system-prompt hash transition). A consumer rolling up `roundtrips.jsonl` to per-turn or per-agent-run totals does so by grouping on these IDs — no client-side reconstruction.

## 2. Acceptance criteria

1. Single-turn capture: 1 `turn_id`, 1 `agent_run_id`.
2. Multi-tool-use turn capture (e.g. `"list /tmp and identify owner and size"`): one stable `turn_id` across all continuation RTs until `stop_reason ≠ tool_use`; one `agent_run_id`.
3. Multi-agent-run capture (prompt that spawns a sub-agent via the `Agent` tool): ≥ 2 distinct `agent_run_id`s within one `session_id`. (Note from [E3](E3-marking-detector-boundary-trace.md): sub-agent invocations share the parent's `x-claude-code-session-id` on the wire — system-prompt-hash transition is the only viable trigger, confirming ADR 023 §2.5's design premise.)
4. Boundary decisions match ADR 023 §2.4 / §2.5 verbatim; any divergence captured in §E3 §A is closed. §2.4 turn-id logic is confirmed correct by E3 against three real captures (1/1/2 turns observed); §2.5 unfireable today (see AC #6).
5. `tap.jsonl` `marks` block extends to carry `agent_run_id` alongside the existing `session_id` / `turn_id` / `parent_session_id`.
6. **Plumb `request_system_hash` to the detector.** Per [E3](E3-marking-detector-boundary-trace.md) §A finding #1, both call sites in `crates/noodle-proxy/src/wirelog.rs` (around `:508` for `on_request_open` and `:1293` for `on_response_close`) currently pass literal `None`. This slice extracts the canonical `system` field from the `/v1/messages` request body, hashes it (`SystemHash`), and passes it to both methods. Without this, §2.5 detection cannot fire even if the minting logic is implemented.

## 3. Abstractions introduced or refined

- **`AnthropicMarkingDetector`** (refined): existing `turn_id` minting validated against §2.4 from the [E3](E3-marking-detector-boundary-trace.md) trace; new `agent_run_id` minting added per §2.5 (system-prompt-hash transition).
- **`MarkingState`** (refined): adds `agent_run_id` alongside `session_id` / `turn_id`.
- **`WireMarks`** + the `tap.jsonl` `marks` block schema: extended additively with `agent_run_id`.

## 4. Patterns applied

- **State machine** — the detector advances a per-session FSM on every observed request; transitions mint new IDs per ADR 023.

## 5. Test plan

- **Unit:** drive `AnthropicMarkingDetector` through each ADR 023 §2.4 case (fresh-session, end_turn, max_tokens, tool_use continuation); assert `turn_id` mints exactly when expected. AC #1, #2.
- **Unit:** drive the same detector through system-prompt-hash transitions; assert `agent_run_id` mints exactly when expected. AC #3.
- **Integration:** new `crates/noodle-proxy/tests/e2e_turn_and_agent_run_ids.rs` exec-claude harness against the three captures from [E3](E3-marking-detector-boundary-trace.md); asserts ACs end-to-end.

One PR. Detector refinement + schema extension + tests. Target: under 500 reviewable lines.

## 7. Out of scope

- `RoundTripRecord` consumes these IDs but doesn't define them — [040.b](040.b-roundtripsink-and-roundtrips-jsonl.md).
- Cross-flow detection (the detector is per-flow today; cross-flow correlation by session is the consumer's job).
- Parallel-sub-agent disambiguation when multiple `Agent` tool calls fire in parallel — flagged as a future story if [E3](E3-marking-detector-boundary-trace.md) surfaces it.
