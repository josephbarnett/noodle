## 1. Value delivered

After this slice ships, every record in `~/.noodle/side_effects.jsonl` carries `event_id` + `turn_id` + `session_id` (full, not 8-char prefix) + `agent_run_id` + a non-zero `at_unix_ms`. A downstream consumer joins `tap.jsonl` ↔ `side_effects.jsonl` by `event_id` and gets 1:1 correspondence per round-trip. This unblocks every cadence slice that follows (040.b, 040.c, 042, 043).

## 2. Acceptance criteria

1. Live `~/.noodle/side_effects.jsonl` records on every variant (`Hint`, `Artifact`, `AuditEvent`, `ResolvedRecord`) carry `event_id`, `turn_id`, `session_id`, `agent_run_id`, `at_unix_ms` after this lands. Baseline from [E1](E1-live-correlation-gap-audit.md) §A goes to 100% for every cell.
2. `session_id` is the full ADR 028 `MarkingSessionId` value, not a hash prefix.
3. `at_unix_ms` is never `0` on disk for any record produced by the proxy.
4. `jq -s 'group_by(.event_id) | map(.[0].event_id)' ~/.noodle/side_effects.jsonl` returns event ids that all appear on `~/.noodle/tap.jsonl`; no orphans.
5. The four fields land additively — no existing field is removed or renamed; `session_prefix` may stay as a derived convenience but `session_id` is the canonical key.
6. Hot-path budget: zero allocation per record beyond the `SmolStr` / `u64` copies; engine `record(...)` remains non-blocking; the writer keeps drop-on-full posture.

## 3. Abstractions introduced or refined

- **`Correlation`** (new, in `noodle-core::layered`): a shared block with the four IDs + `at_unix_ms`. Either embedded as a struct field on each SideEffect variant or flattened. Decision goes on the implementing PR.
- **`Hint` / `Artifact` / `AuditEvent` / `ResolvedRecord`** (refined): each grows the correlation block.
- **`EngineState`** (refined): the drain path (`finish` in `wirelog.rs`) stamps correlation from the `MarkingState` + `request_id` it already holds.
- **`SideEffectsJsonlSink`** (refined): serialiser extends `JsonlEntry` variants with the four fields.
- **`ResolvedRecord.session`** (refined): drop the ephemeral `Session` minted at drain; build from `MarkingState.marks.session_id`.

DI seam: the `Correlation` builder takes a `Clock` for `at_unix_ms` so tests substitute a `FakeClock`.

## 4. Patterns applied

- **Decorator** — correlation is stamped onto every SideEffect as it leaves the engine; the engine seam decorates the inner record without changing its semantics.

## 5. Test plan

- **Unit:** schema golden tests pin every variant's serialised form against fixtures with non-zero correlation. ACs #1–#5 mapped.
- **Unit:** stamping is bypass-resistant — synthesise a SideEffect at drain time without correlation and assert the sink rejects / fills it.
- **Property:** flood the sink with N synthetic flows; assert peak memory bounded; hot-path drop counter increments before the engine blocks. AC #6 mapped.
- **Integration:** new `crates/noodle-proxy/tests/e2e_side_effects_correlate_with_tap.rs` drives `claude -p` through the release binary; asserts every `side_effects.jsonl` record's `event_id` joins to a `tap.jsonl` record. ACs #1, #4 mapped end-to-end.

## 6. PR scope

One PR. Schema additions + engine drain stamp + serialiser + golden tests + new e2e. Target: under 600 reviewable lines.
