## 1. Value delivered

After this slice ships, a downstream consumer (the downstream telemetry consumer primary; any OTel-shape consumer by generalisation) tails **one file**, `~/.noodle/roundtrips.jsonl`, and reads **one self-contained record per LLM round trip** — request meta + response meta + extracted markers + Resolved attribution + token usage + latency + the contributing Hints / Artifacts / Audits — already correlated by the four IDs. No client-side reconstruction. This is the data-shape proof point of the macOS-parity cadence.

## 2. Acceptance criteria

1. `~/.noodle/roundtrips.jsonl` written by the proxy at flow finish. One JSONL line per completed HTTP round trip.
2. Record shape matches ADR 023 §4 verbatim.
3. Every record carries all four correlation IDs from [040.a](040.a-side-effect-correlation-block.md).
4. `jq -s 'length' ~/.noodle/roundtrips.jsonl` equals the count of `/v1/messages` response records on `~/.noodle/tap.jsonl` for the same capture window.
5. `jq '.flow_id' ~/.noodle/roundtrips.jsonl` joins 1:1 against `tap.jsonl` response records by `flow_id` (or `event_id`).
6. Per-flow buffer in `RoundTripSink` is bounded; total memory bounded by `concurrent_flows × small_constant`.
7. Hot-path posture: same drop-on-full as `SideEffectsJsonlSink`; engine never blocks on the sink.
8. `TapUsage` widens to sibling-level `service_tier` / `inference_geo` / `cache_creation.{ephemeral_5m_input_tokens, ephemeral_1h_input_tokens}` so the `ai-telemetry` v0.0.2 mapper (slice 042) reads them directly rather than from `vendor_extras`. [E2](E2-ai-telemetry-fixture-mapping.md) §A confirms the schema requires them as direct siblings of `tokens`. Tap-format golden test pinned.

## 3. Abstractions introduced or refined

- **`RoundTripRecord`** (new, `noodle-core::layered` or a sibling module). ADR 023 §4 pins the field list. Strongly typed; serde-serialisable.
- **`RoundTripSink`** (new, `noodle-adapters::sink`): a `SideEffectSink` impl that buffers per-flow `Hint` / `Artifact` / `AuditEvent` indexed by `flow_id`; assembles the `RoundTripRecord` on `ResolvedRecord` arrival (or explicit flow-end notification); writes one JSONL line; drops the buffer.
- Composes alongside `SideEffectsJsonlSink` + `TracingSink` under `MultiSideEffectSink`. Sibling sinks fed from the same `SideEffect` stream.

DI seam: `RoundTripSink::spawn(path, clock, capacity)` — `Clock` trait substitutes a `FakeClock` in tests.

## 4. Patterns applied

- **Aggregator** — per-`flow_id` buffering until `Resolved` arrives; then assemble + emit + drop. Symmetric to `EventsAccumulator` on the body tee.
- **Composite** — sibling sinks under `MultiSideEffectSink`.

## 5. Test plan

- **Unit:** `RoundTripSink` against `FakeClock` + in-memory writer. Synthesise one flow's SideEffect stream; assert exactly one line on `Resolved` arrival; buffer dropped; schema matches ADR 023 §4. ACs #1, #2.
- **Unit:** out-of-order Resolved arrival edge case (Resolved before all Hints) — sink must assemble correctly.
- **Property:** N concurrent flows × M SideEffects each; bounded memory; drop counter on saturation. AC #6, #7.
- **Integration:** new `crates/noodle-proxy/tests/e2e_roundtrips_match_tap.rs` drives `claude -p` through the release binary; asserts ACs #4, #5.
